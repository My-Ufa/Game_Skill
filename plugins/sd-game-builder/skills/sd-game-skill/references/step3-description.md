# Шаг 3 — Описание игры

## Цель

Собрать единый документ-описание СД-игры по 11 блокам.
Он станет основой для генерации экономики, плацдарма, ролевых листов и установки.

## Структура описания

### 1. Контекст и ситуация
Отрасль, регион, временной горизонт, ключевая проблема/возможность рынка.

### 2. Основной конфликт
Что сталкивается: интересы каких сторон, какой ресурс ограничен,
чьё положение под угрозой.

### 3. Цепочка создания продукта
Производство → Дистрибуция → Конечный потребитель (или вариант под отрасль).
**Важно:** цепочка может быть укороченной — одна роль способна производить
продукт целиком. Не вытягивай звенья искусственно.

### 4. Механика рынка (как разыгрывается спрос)

Зафиксируй одно правило распределения клиентов/заказов:

| Правило | Описание | Когда уместно |
|---------|----------|---------------|
| По скорости | Кто первым подал заявку — получает клиента (FIFO) | Дефицитный рынок, скоропортящийся спрос |
| По цене | Победитель — наименьшая цена | Товарный рынок, B2B тендеры |
| По качеству/мощности | Победитель — наибольший показатель качества или мощность актива | Рынок с дифференциацией |
| По доле рынка | Спрос делится пропорционально текущим долям | Зрелый стабильный рынок |
| Смешанный | Например: сначала фильтр по лицензии, затем по цене | Регулируемые отрасли |

Также опиши:
- Что происходит при превышении спроса (очередь? потеря клиентов? скидка?)
- Что происходит при нехватке предложения (дефицит → рост цены? регулятор?)

### 5. Действия игроков (что можно делать за такт)

Для каждого действия укажи:
- **Название** — понятное игрокам
- **Стоимость** (у.е.) или условие исполнения
- **Эффект** — что изменится в следующем такте

Минимальный набор действий:
```
• Подать заявку на рынок → получить клиентов по правилу рынка
• Купить игровой предмет (завод/офис/точку продаж/персонал)
• Купить технологию у тех.компании
• Нанять / уволить персонал
• Увеличить мощности существующего актива
• Взять кредит у банка / вернуть кредит
• [Регулятор] Подать постановление, выдать / отозвать лицензию, выписать штраф
• [Игротехник] Объявить шок, изменить спрос, запустить событие
```

### 6. Игровые предметы (активы игроков)

Для каждого предмета — все четыре параметра обязательны:

| Предмет | Стоимость покупки (у.е.) | Расходы/такт (у.е.) | Мощность (ед./такт) | Условия |
|---------|-------------------------|---------------------|---------------------|---------|
| Завод | 500 | 80 | 200 | любой Бизнес |
| Офис | 200 | 40 | 100 | любой |
| Точка продаж | 150 | 30 | 80 | нужна лицензия |
| Персонал (синие) | 50 | 20 | 50 | лимит 10 чел. на роль |
| Персонал (белые) | 80 | 30 | 0 (управление) | ... |

Типы предметов: завод / офис / точка продаж / склад / транспорт /
персонал-белые / персонал-синие / лицензия / другое.

### 7. Технологии

Для каждой технологии — все параметры обязательны:

| Параметр | Что указать |
|----------|-------------|
| Название | Конкретное, понятное |
| Эффект | «снижает себестоимость на X у.е./ед.» / «+X у.е. средний чек» / «приоритет при разыгрывании» / «сокращает N чел. → −X у.е./такт» / «открывает сегмент объёмом X у.е.» |
| Стоимость разработки | у.е. разово (платит разработчик — обычно тех.компания) |
| Стоимость внедрения | у.е. разово (платит покупатель) |
| Расходы/такт | у.е. каждый такт после внедрения |
| Кто может внедрять | «только тех.компания» / «любой игрок» / «только игротехник» |

### 8. Сценарии и события
Что может произойти за 3 такта: внешние шоки, действия регулятора,
выход технологии, изменение спроса.

### 9. Временной горизонт
Обычно 3 такта. 1 такт = N лет — так, чтобы был виден эффект инвестиций.

### 10. Гипотезы для проверки игрой
2–4 вопроса формата «Что произойдёт, если…».

### 11. Победа / результат
Как оценивается успех каждой стороны.

## Правила ведения диалога

- Задавай по 1–2 уточняющих вопроса за раз.
- **ОБЯЗАТЕЛЬНО в каждом ответе** — блок «❓ Открытые вопросы»:
  конкретные пробелы, противоречия, неясные параметры. Не общие, а адресные.
- После каждого ответа обновляй GAME_UPDATE.
