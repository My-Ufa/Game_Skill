# Шаг 5 — Экономика игры (→ Excel)

## Цель

Рассчитать экономику по каждой роли и сгенерировать Excel-таблицу
с балансом на 3 такта в двух сценариях (минимум / максимум).

## Алгоритм расчёта

### 1. Единица измерения
Сначала считай в рублях по реальным данным из шага 2.
Выбери делитель: 1 000 / 100 000 / 1 000 000 / 100 000 000 / 1 000 000 000.
Зафиксируй: **«1 у.е. = X ₽»**.

### 2. Рынок
- Объём рынка (у.е.)
- Цена, по которой клиенты покупают продукт (у.е./ед.)

### 3. По каждой роли

| Параметр | Описание |
|----------|----------|
| Стартовый счёт | сколько у.е. на начало игры |
| Доходы за такт | источники: продажи, бюджет, дотации (с условиями), проценты |
| Постоянные расходы/такт | зарплаты, аренда, операционные |
| Разовые расходы | стройка, найм, R&D, покупка технологии |
| Мощность | сколько производит/обслуживает за такт; от чего зависит |

### 4. Проверка баланса

Правило трёх тактов при **бездействии**:
- Такт 1 → конец: плюс (никто не уходит в минус сразу)
- Такт 2 → конец: ноль или небольшой плюс
- Такт 3 → конец: **минус** (давление на действие — иначе игра неинтересна)

Если не сходится — добавь денег на стартовый счёт или пересчитай расходы.
Называй конкретное слабое место: «Роль X на такте 2 уходит в −500 при бездействии».

### 5. Сценарная таблица (это и есть Excel)

Построй таблицу: **все роли × 3 такта × 2 сценария (MIN / MAX)**

```
Роль          | Сценарий | Такт 1          | Такт 2          | Такт 3
              |          | Доход/Расход/Баланс | ...          | ...
Сбербанк      | MIN      |  500 / 800 / 200 | 500 / 800 / -100 | ...
Сбербанк      | MAX      | 1200 / 900 / 500 | 1500 / 950 / ...  | ...
```

- **MIN** = игрок бездействует (только базовые доходы и постоянные расходы)
- **MAX** = игрок действует оптимально (вкладывается в технологии, нанимает,
  строит, использует все механики)

Внизу таблицы: общая сумма денег в игре, мин/макс заработок каждой роли.

### 6. Поиск узких мест

После таблицы явно укажи проблемы:
- Кто уходит в минус на такте 1 при MIN → нужен больший стартовый счёт
- Кто безубыточен даже при MAX → нет стимула действовать
- Кто сразу богатеет при MIN → дисбаланс

### 7. Экономика технологий

| Параметр | Значение |
|----------|----------|
| Стоимость разработки | у.е. |
| Стоимость внедрения | у.е. |
| Затраты каждый такт | у.е./такт |
| Эффект | ↓ затраты / ↓ персонал / ускорение / новое качество / расширение рынка |
| Окупаемость | на следующий такт после внедрения ≈ 0; через такт — прибыль |

## Блок «❓ Открытые вопросы для уточнения»

Обязательно в конце — конкретные пробелы:
- Какой источник дохода у роли X? (не указано)
- Какова ставка дотации для регулятора? (нет данных)
- Сколько мощности даёт один завод? (нужно для расчёта MAX-сценария)
