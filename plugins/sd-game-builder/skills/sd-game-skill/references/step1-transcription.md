# Шаг 1 — Транскрибация встречи

## Что делаешь

Пользователь прикладывает аудио, видео или сырой текст встречи.
Твоя задача — очистить, структурировать и извлечь всё полезное для сборки игры.

## Алгоритм обработки

1. **Очистка** — убери слова-паразиты, повторы, разговорный шум. Оставь смысл.
2. **Спикеры** — раздели по участникам, если они различимы (`Спикер А:`, `Спикер Б:`).
3. **Темы** — структурируй по блокам с заголовками (`## Тема`).
4. **Ключевые блоки** (вынеси отдельно):
   - Решения и договорённости
   - Открытые вопросы и риски
   - Цифры, названия компаний, продукты, сроки
   - Гипотезы о рынке / отрасли / игре

## Обязательная пост-структуризация

После очистки — структурируй материал по блокам СД-игры.
Заполняй только то, что **прямо звучало** в транскрипте. Пустое → «—».

```
## Описание
(отрасль, регион, временной горизонт, ключевая проблема)

## Конфликт
(что сталкивается, за что борьба, чьё положение под угрозой)

## Механики
(что участники могут делать: покупать, строить, нанимать, кредитовать и т.п.)

## Экономика
(любые цифры: цены, объёмы, затраты, обороты, маржа)

## Роли
(упомянутые компании, типы участников, кто с кем взаимодействует)

## Гипотезы
(что хотят проверить игрой, «что будет если…»)
```

## Резюме и следующий шаг

Заверши резюме 3–5 пунктов.
Предложи: «Эти материалы пригодятся для шага 3 (Описание игры).
Хочешь сразу перейти к шагу 2 (Анализ рынка) или к шагу 3?»

## GAME_UPDATE после этого шага

Заполни поля, которые явно прозвучали в транскрипте:
`title`, `context`, `conflict`, `hypotheses`, `roles` (только имена/теги если упоминались).
