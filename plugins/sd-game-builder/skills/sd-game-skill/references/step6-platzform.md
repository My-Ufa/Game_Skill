# Шаг 6 — Плацдарм (→ PPTX или Figma, 1 лист A4)

## Цель

Создать визуальную схему игры на 1 лист A4 — плацдарм.
Показывает всех участников, их связи и ключевые цифры.

## Расположение блоков на листе

```
┌─────────────────────────────────────────────────────────┐
│ Название игры (левый верх, крупно, жирно)               │
├─────────────────────────────────────────────────────────┤
│          [РЕГУЛЯТОРЫ] (красная обводка)                 │
│  ─────────────────────────────────────────────────────  │
│  [БАНКИ]           [центр: основной бизнес]  [ТЕХ.КОМ] │
│  (фиолетовая)      Поставщик→Произв.→Дистр.  (синяя)   │
│                    (зелёная/нейтральная)                 │
│                                                          │
│          [КЛИЕНТЫ] (внизу, иконка человека)             │
│          Такт 1 | Такт 2 | Такт 3 (если меняется объём) │
├─────────────────────────────────────────────────────────┤
│ [Механики] (правый нижний угол)                         │
└─────────────────────────────────────────────────────────┘
```

## Карточка каждой роли

Прямоугольник со скруглёнными углами:
- **Название** — 16px, жирно
- 💰 + расчётный счёт на начало игры
- «Расходы в такт:» + сумма (у.е.)
- Активы: персонал / заводы / офисы / точки продаж

Цвет обводки: Регулятор=красная, Банк=фиолетовая, Тех.компания=синяя, Бизнес=зелёная.

## Технологические компании — расширенная карточка

У тех.компаний добавить:
- Краткое описание технологии/механики
- Стоимость разработки и внедрения (у.е.)

## Блок «Механики» (правый нижний угол)

Короткие описания ключевых механик игры (4–6 строк).

## Что проверить перед генерацией

- Данные по всем ролям заполнены (счёт, расходы, активы)?
- Цепочка стрелками показана (если она есть)?
- Клиенты размещены снизу?
- Есть ли таблица по тактам для клиентов (если объём меняется)?

## Выбор формата перед генерацией

Перед генерацией спроси пользователя:

> «В каком формате генерировать плацдарм?
> 1. **PPTX** — файл PowerPoint, удобно распечатать или вставить в презентацию
> 2. **Figma** — интерактивный макет, удобно редактировать и делиться ссылкой
> 3. **Оба** — сначала Figma, потом экспортируем в PPTX»

### Если выбрана Figma

Используй MCP-инструмент `use_figma` для создания плацдарма:
- Создай фрейм A4 (794×1123 px) или как отдельный экран
- Каждая роль — компонент-карточка с цветовой обводкой по типу (Регулятор=красный, Банк=фиолетовый, Тех.компания=синий, Бизнес=зелёный)
- Стрелки между ролями — коннекторы
- Блок «Механики» — текстовый фрейм в правом нижнем углу
- После создания дай ссылку на файл Figma
