# Шаг 7 — Ролевые листы (→ PPTX, A4, 1 лист = 1 роль)

## Цель

Создать индивидуальный игровой лист для каждой роли.
Игрок получает этот лист — и знает всё, что ему нужно для игры.

## Структура каждого листа

```
┌─────────────────────────────────────────────┐
│ НАЗВАНИЕ РОЛИ (крупно, жирно)    [Легенда]  │
│                                  2–3 строки  │
│                                  о функции   │
├─────────────────────────────────────────────┤
│ 💰 Расчётный счёт: XXX у.е.                 │
│                                              │
│ Активы:                                      │
│ • Персонал: N чел.                           │
│ • Заводы / Офисы: N                          │
│ • Технологии: (список)                       │
├─────────────────────────────────────────────┤
│ КЛЮЧЕВЫЕ ДЕЙСТВИЯ (что можно делать за такт) │
│ 1. ...                                       │
│ 2. ...                                       │
├─────────────────────────────────────────────┤
│ ЭКОНОМИКА                                    │
│ Доходы: источник 1 — XXX у.е./такт           │
│ Постоянные расходы: YYY у.е./такт            │
│ Улучшения (разовые): ZZZ у.е.                │
├─────────────────────────────────────────────┤
│ [СПЕЦ-БЛОК по типу роли — см. ниже]         │
└─────────────────────────────────────────────┘
```

## Специальные блоки по типу роли

### Технологическая компания → «Технологии»
```
Технология X:
  Разработка: XXX у.е. (разово)
  Внедрение: YYY у.е. (разово, у покупателя)
  Затраты/такт: ZZZ у.е.
  Эффект: [↓ затраты на 20%] / [↑ мощность] / [новое качество] / ...
```

### Регулятор → «Инструменты воздействия»
```
• Постановление: влияет на [механику X], принимается за [стоимость/условие]
• Штраф: до XXX у.е. за нарушение [правила Y]
• Лицензия: выдаётся роли [Z], стоит XXX у.е., даёт право на [действие]
```

### Банк / Инвестор → «Кредитование»
```
• Кредит: до XXX у.е., ставка N%/такт, условие возврата
• Инвестиции: от XXX у.е., доля / условия
• Лимит на одного заёмщика: XXX у.е.
```

## Что проверить по каждой роли перед генерацией

- Стартовый счёт заполнен?
- Все источники дохода прописаны?
- Постоянные расходы известны?
- Стоимость улучшений (разовые затраты) есть?
- Спец-блок соответствует типу роли?
